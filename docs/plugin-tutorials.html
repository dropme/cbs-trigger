Writing plugins for Triggerd

A hands-on series that builds from a one-line button to custom nodes, on-screen visuals, and show automation. No build step, no framework setup — a plugin is one JavaScript file and a small manifest.

Before you begin

A plugin adds your own nodes, transport buttons, commands, and on-screen layers (captions / subtitles / visuals) to the app. It is plain CommonJS JavaScript that the app evaluates with module, exports, host, and React already in scope — so you never install or bundle anything.

Trust
Plugins run in-process with full trust — they are real code loaded into the app, like a browser extension or a VST. Only install plugins you trust.

Where plugins live

Each plugin is a folder containing a manifest.json and its main script. The app scans two locations at startup:

  • <userData>/plugins/ — the per-user, writable location that survives app updates. On Windows: %APPDATA%/sound-pattern-player/plugins/; macOS: ~/Library/Application Support/…; Linux: ~/.config/….
  • <app>/plugins/ — a folder next to the app, handy while developing.

The quickest way to find it: Settings → Plugins → Open plugins folder.

The develop loop

  1. Copy the ready-made plugins/plugin-template/ folder and rename it.
  2. Edit index.js.
  3. Click Settings → Plugins → Reload plugins — your changes apply without relaunching the app.
  4. Open DevTools (Ctrl/Cmd+Shift+I) to see [plugin] log messages. A throw in your plugin is caught and logged — it never crashes the app.
Editor autocomplete
Ship a jsconfig.json next to your index.js (the template has one) and add the JSDoc @typedef imports from plugins/triggerd-plugin.d.ts. Your editor then autocompletes the whole host API and type-checks your plugin — still with no build step.

Anatomy of a plugin

manifest.json
{
  "id": "com.example.myplugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "apiVersion": 5,
  "main": "index.js"
}

apiVersion is the lowest host version whose features you use. The host runs plugins targeting its version or older, and skips (with a log) anything newer. The current host is v5. Baseline is v1; host.http arrived in v2, host.http.request in v3, host.mixer.fade + ctx.hold() in v4, and host.transport / vars / lighting / timecode in v5.

index.js
/* global module, host, React */

module.exports = {
  id: 'com.example.myplugin',
  name: 'My Plugin',
  version: '1.0.0',
  apiVersion: 5,

  activate(host) {
    // register your nodes / buttons / commands here
  }
}

activate(host) runs once when the plugin loads. host is the only surface you touch — a stable façade over the app. Build UI with React.createElement (the React argument is the host's own React, so hooks work and you need no bundler).

1 · Hello, transport button the simplest plugin

Goal: add a button to the play-bar transport that shows a toast when clicked.
  1. Create hello/manifest.json and hello/index.js (or copy the template).
  2. In activate, call host.registerTransportControl with an id, a label, and a run() callback.
  3. Drop the folder in the plugins directory and click Reload plugins.
index.js
/* global module, host, React */

module.exports = {
  id: 'com.example.hello',
  name: 'Hello',
  version: '1.0.0',
  apiVersion: 1,

  activate(host) {
    host.registerTransportControl({
      id: 'com.example.hello.btn',
      label: '👋 Hi',
      title: 'Say hello',          // tooltip (optional)
      run() {
        host.ui.toast('Hello from my plugin!')
      }
    })
  }
}

That's a complete plugin. A few extras on transport controls:

  • isActive() — return a boolean to make the button light up (for a toggle).
  • when: 'running' | 'idle' | 'always' — show the button only while a show is running, only while idle, or always (default).
Note
Every register* call returns a Disposable. You can ignore it, or keep it and call .dispose() from an optional deactivate() to unregister cleanly.

2 · Your first node card + behaviour

Goal: a node you can drop on the canvas that, when the playhead enters it, bumps a show variable and toasts the new count.

A node type bundles its palette entry, default data, canvas card, optional inspector, runtime behaviour, and optionally an on-screen layer. The minimum is a palette entry, default data, a card, and an onEnter.

  1. Write a Card component — the inner content of the canvas card (the app draws the frame and the input/output handles for you). Reuse the app's classes spp-node-head and spp-node-name for a native look.
  2. Register the node with host.registerNodeType: give it a namespaced id, a palette icon/label, a defaultData() factory, the card, and ports.
  3. Add onEnter(node, ctx) — the side effect that runs when the playhead reaches the node.
index.js
/* global module, host, React */

function Card({ node }) {
  return React.createElement(
    'div', { className: 'spp-node-head' },
    React.createElement('span', { className: 'spp-node-name' },
      '✨ ' + (node.name || 'Counter'))
  )
}

module.exports = {
  id: 'com.example.counter', name: 'Counter', version: '1.0.0', apiVersion: 5,
  activate(host) {
    host.registerNodeType({
      id: 'com.example.counter.node',
      palette: { icon: '✨', label: 'Counter' },
      defaultData: () => ({ varName: 'count' }),
      card: Card,
      ports: { input: true, output: true },
      onEnter: (node, ctx) => {
        const name = String(node.data.varName || 'count')
        const next = host.vars.get(name) + 1
        host.vars.set(name, next)
        ctx.log('counter', name, '→', next)
        host.ui.toast(name + ' = ' + next)
      }
    })
  }
}

Reload, then in Edit mode find ✨ Counter in the palette on the left, drop it in, wire it into a chain, and press Play. Each time the playhead enters it, the count variable increases and a toast appears.

How advancing works
onEnter is fire-and-forget: the node runs its side effect, then the playhead advances along its outgoing connection per that connection's own trigger (auto-advance or wait-for-key) — exactly like the built-in OSC / FX nodes. It does not block. (Tutorials 6 and 7 show how to hold or branch.)
ctx vs host
Inside onEnter you get a ctx with sendMidi, sendOsc, playFile, log, getState, and hold(). Prefer ctx.sendMidi/sendOsc over host.midi/osc for anything a cue "emits", because ctx sends respect Rehearsal mode (they're muted while rehearsing); direct host sends are not.

3 · An inspector editable node data

Goal: let the operator edit which variable the Counter bumps, from the inspector side-panel.

Add an Inspector component. It renders below the shared Name / Colour / Tags block and receives helpers to mutate the node:

PropWhat it does
nodeThe current node (node.data is your persisted state).
updateData(patch)Shallow-merges patch into node.data.
updateNode(patch)Set name, color, tags, or operatorNote.
remove()Delete the node.
index.js — add this component and reference it as inspector
function Inspector({ node, updateData }) {
  const varName = String(node.data.varName || 'count')
  return React.createElement(
    'label', { className: 'spp-field' },
    React.createElement('span', null, 'Variable name'),
    React.createElement('input', {
      type: 'text',
      value: varName,
      onChange: (e) => updateData({ varName: e.target.value })
    })
  )
}

// …then in registerNodeType({ … }) add:
//   card: Card, inspector: Inspector,

Reuse the app's form classes so it looks native: spp-field (a labelled control), spp-field-row (controls side by side), spp-file-row + spp-file-name (a chosen-file row), spp-hint-small (a muted help line), and spp-check (a checkbox row).

Persistence
node.data is saved and loaded with the project. If a project opens before your plugin has loaded, the node keeps its kind and data and re-renders as your node once it registers — so old projects never lose data.

4 · An on-screen layer captions & visuals

Goal: a node that shows a caption over the output while it is the current cue — visible in the Play-Mode preview and on the Show-Mode / audience display.

Add a layer to the node type. When the playhead enters the node it becomes the current node for that layer (last-wins, like the built-in image/video and text nodes) and your render component is drawn over the media until another node takes the layer. Use kind: 'text' for captions/subtitles or kind: 'visual' for a full-bleed background. The layer is full-bleed and click-through — position your content with absolute CSS.

index.js
function Card({ node }) {
  return React.createElement('div', { className: 'spp-node-head' },
    React.createElement('span', { className: 'spp-node-name' }, '💬 ' + (node.name || 'Caption')))
}

function Inspector({ node, updateData }) {
  return React.createElement('label', { className: 'spp-field' },
    React.createElement('span', null, 'Caption text'),
    React.createElement('textarea', {
      className: 'spp-textarea', rows: 2,
      value: String(node.data.text || ''),
      onChange: (e) => updateData({ text: e.target.value })
    }))
}

function Caption({ node }) {
  const text = node.data.text
  if (!text) return null
  return React.createElement('div',
    { style: { position: 'absolute', left: 0, right: 0, bottom: '8%', textAlign: 'center' } },
    React.createElement('span', { style: {
      background: 'rgba(0,0,0,0.72)', color: '#fff',
      font: '500 3.6vmin system-ui', padding: '0.15em 0.5em', borderRadius: '0.25em'
    } }, text))
}

module.exports = {
  id: 'com.example.caption', name: 'Caption', version: '1.0.0', apiVersion: 1,
  activate(host) {
    host.registerNodeType({
      id: 'com.example.caption.node',
      palette: { icon: '💬', label: 'Caption' },
      defaultData: () => ({ text: '' }),
      card: Card, inspector: Inspector,
      ports: { input: true, output: true },
      layer: { kind: 'text', render: Caption }
    })
  }
}
Animate against the clock
To animate over time, read host.playback from a requestAnimationFrame loop inside your layer component. host.playback.getProgress().elapsed is the current clip/video position in seconds (resets each cue — how .srt files are authored); host.playback.getElapsedMs() is milliseconds since the show started (a single show clock). Tutorial 5 uses this pattern.

5 · Async & HTTP fetch on enter

Goal: a node that fetches a joke over HTTP each time the playhead enters it and shows it on the text layer — even on a loop.

Two new ideas here. First, host.http.request v3 does real HTTP from the app's main process — no CORS — and never rejects (a network error resolves as { ok: false, status: 0 }). Second, a layer component stays mounted while its node is current, so it can't re-fetch on its own each loop; instead, drive the fetch from onEnter and push the result into a tiny per-node store the layer subscribes to.

index.js
/* global module, host, React */
var URL = 'https://v2.jokeapi.dev/joke/Any?format=json'

// A per-node store so a fetch triggered on enter reaches the (already-mounted) layer.
var store = {}, listeners = {}
function emit(id, val) {
  store[id] = val
  ;(listeners[id] || []).forEach((fn) => fn(val))
}
function fetchJoke(id) {
  emit(id, { state: 'loading' })
  host.http.request({ url: URL }).then((res) => {
    if (!res.ok) return emit(id, { state: 'error' })
    var d = JSON.parse(res.body)
    emit(id, { state: 'ok', text: d.joke || (d.setup + ' … ' + d.delivery) })
  })
}

function Layer({ node }) {
  var st = React.useState(store[node.id] || { state: 'loading' })
  var joke = st[0], setJoke = st[1]
  React.useEffect(() => {
    var set = listeners[node.id] || (listeners[node.id] = [])
    set.push(setJoke)
    if (!store[node.id]) fetchJoke(node.id)     // first mount (or the presentation window)
    return () => { listeners[node.id] = set.filter((f) => f !== setJoke) }
  }, [node.id])
  if (joke.state !== 'ok') return null
  return React.createElement('div',
    { style: { position: 'absolute', inset: 0, display: 'flex', alignItems: 'center',
      justifyContent: 'center', padding: '0 8%', textAlign: 'center' } },
    React.createElement('div', { style: { color: '#fff', font: '600 4.4vmin system-ui' } }, joke.text))
}

module.exports = {
  id: 'com.example.joke', name: 'Joke', version: '1.0.0', apiVersion: 3,
  activate(host) {
    host.registerNodeType({
      id: 'com.example.joke.node',
      palette: { icon: '😂', label: 'Joke' },
      defaultData: () => ({}),
      card: ({ node }) => React.createElement('div', { className: 'spp-node-head' },
        React.createElement('span', { className: 'spp-node-name' }, '😂 ' + (node.name || 'Joke'))),
      ports: { input: true, output: true },
      onEnter: (node) => fetchJoke(node.id),    // fires on every visit, incl. a goto loop
      layer: { kind: 'text', render: Layer }
    })
  }
}

Wire it into a loop (a Go to back to it, or a Delay + Go to) and every pass fetches a fresh joke. This is the shipped plugins/joke-teller example, trimmed.

6 · A “wait for X” node holding the playhead

Goal: a node that holds the show until an OSC message arrives, then advances — like the built-in Wait node, but yours.

Call ctx.hold() v4 synchronously inside onEnter. It keeps the playhead on your node instead of advancing and returns a release() you call later — from your own event or timer — to advance along the node's output. Holds are cleaned up automatically on Stop / Reset / Panic.

index.js — the onEnter
onEnter: (node, ctx) => {
  const release = ctx.hold()                 // keep the playhead here
  const want = String(node.data.address || '')
  const sub = host.osc.onMessage((msg) => {
    if (want && msg.address !== want) return  // blank = release on any OSC
    sub.dispose()                          // stop listening
    release()                              // advance now
  })
}
Two rules
1. Call ctx.hold() synchronously in onEnter (not after an await) or the playhead will have already advanced. 2. Always dispose() the listener you created when you release, so a re-entry doesn't stack listeners.

Prefer a fixed duration instead of an event? Return seconds from duration(node) and the node holds the playhead for that long — like a clip — driving the transport progress bar while it runs. Return 0 for an instant pass-through. (ctx.hold() overrides duration().)

7 · A branch node multiple outputs

Goal: a node with two outputs — heads and tails — that randomly picks one each time the playhead reaches it.

Declare named ports.outputs (each gets its own output handle on the card) and a route(node, ctx) that returns which output id the playhead should leave by (or null to end the lane). This is how the built-in Repeat's loop/done and Condition's true/false work.

index.js
host.registerNodeType({
  id: 'com.example.coin.node',
  palette: { icon: '🪙', label: 'Coin flip' },
  defaultData: () => ({}),
  card: ({ node }) => React.createElement('div', { className: 'spp-node-head' },
    React.createElement('span', { className: 'spp-node-name' }, '🪙 ' + (node.name || 'Coin'))),
  ports: {
    input: true,
    outputs: [ { id: 'heads', label: 'Heads' }, { id: 'tails', label: 'Tails' } ]
  },
  route: (node, ctx) => (Math.random() < 0.5 ? 'heads' : 'tails')
})
Round-robin & counters
For a repeating pattern (e.g. take output A three times, then B), keep the per-node counter in host.storage keyed by node.id — route() should stay a pure decision that reads and updates your own state, not the node's persisted data.

8 · A pad / sampler triggered off the playhead

Goal: a node whose sample fires from a keyboard key or a MIDI note at any time — independent of the playhead — like a drum pad.

A node doesn't have to run from the playhead. In activate, register a global listener (host.keyboard.onKey or host.midi.onMessage); when it fires, look up your node instances with host.nodes.ofType('your.kind') and fire their audio with host.audio.play / host.audio.playSlice.

index.js
module.exports = {
  id: 'com.example.pad', name: 'Pad', version: '1.0.0', apiVersion: 1,
  activate(host) {
    host.registerNodeType({
      id: 'com.example.pad.node',
      palette: { icon: '🥁', label: 'Pad' },
      defaultData: () => ({ path: '', name: '', key: 'a' }),
      card: ({ node }) => React.createElement('div', { className: 'spp-node-head' },
        React.createElement('span', { className: 'spp-node-name' },
          '🥁 ' + (node.name || 'Pad') + ' [' + node.data.key + ']')),
      inspector: ({ node, updateData }) => React.createElement('button', {
        onClick: async () => {
          const f = await host.files.pickAudio()
          if (f) updateData({ path: f.path, name: f.name })
        }
      }, node.data.name || 'Choose sample…'),
      ports: { input: false, output: false }   // this pad isn't in the chain
    })

    // Global key listener: fire the matching pad's sample (skipped while typing in a field).
    host.keyboard.onKey((e) => {
      host.nodes.ofType('com.example.pad.node').forEach((n) => {
        if (n.data.key === e.key && n.data.path) host.audio.play(n.data.path, { lane: n.id })
      })
    })
  }
}
Lanes & choke
Pass a lane so a re-trigger cuts the previous hit; host.audio.stop(lane) stops it. For slicing one file into many pads use host.audio.playSlice(path, start, end, opts) with a chokeKey so pads in a group cut each other off. This is the shipped example-toolkit Pad-bank pattern.

9 · Driving the show transport, variables, lighting, timecode

Goal: tour the v5 capabilities that let a plugin control the whole performance.

Transport

host.transport.go()        // start if idle, else advance the current cue (like GO)
host.transport.pause()     // / resume() / stop() / panic()
host.transport.blackout()  // omit the arg to toggle; blackout(true/false) to set
host.transport.goto('Encore')   // jump to the first cue named "Encore"; true if found

Show variables (shared with the Variable / Condition nodes)

host.vars.set('act', 2)
const act = host.vars.get('act')
host.vars.subscribe((name, value) => ctx.log(name, 'changed to', value))

Lighting — Art-Net / DMX

Enable and address Art-Net in Settings → Lighting first. Channels are 1-based, values 0–255.

host.lighting.set(1, [ { channel: 1, value: 255 }, { channel: 2, value: 128 } ], 500)  // fade over 500ms
host.lighting.blackout(1000)   // fade all channels to 0 over 1s

Incoming MIDI Time Code

const tc = host.timecode.get()   // { hours, minutes, seconds, frames } | null
host.timecode.subscribe((tc) => ctx.log(tc.hours, tc.minutes, tc.seconds, tc.frames))
Put it together
A transport button that reads host.timecode and calls host.transport.goto() at set timecodes is a complete “chase an external deck” plugin in a few lines. Or subscribe to host.vars and drive host.lighting to make the rig react to your show's state.

Tips & gotchas

  • Use the host's React. Build all UI with the React argument (or host.React). Bundling your own copy breaks hooks across two Reacts.
  • No build step. Author plain CommonJS with React.createElement. If you insist on JSX, precompile it yourself and ship the output as main.
  • Namespace every id. Node kinds, command ids, and transport ids are global — use com.you.thing.
  • Clean up. If you keep the Disposable from a register*/on* call, dispose it in deactivate().
  • Set apiVersion to the lowest you need — the host runs your version or older and skips newer ones with a log.
  • Errors are isolated. A throw is caught and logged (DevTools → [plugin]); it won't crash the app or other plugins.
  • Rehearsal mode mutes MIDI/OSC a cue emits via ctx, but not direct host.midi/host.osc calls — choose accordingly.
  • Iterate with Reload plugins (Settings → Plugins) — no relaunch. Start from plugins/plugin-template/.
